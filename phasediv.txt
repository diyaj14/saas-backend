## Phase 1 – YouTube integration & basic dashboard (Week 1–3)

### Dev A – Backend: YouTube data & APIs

1. **YouTube OAuth**
   - Implement `/auth/youtube/connect` + callback.  
   - Store refresh tokens encrypted.

2. **YouTube fetch job**
   - Worker job:  
     - On connect → fetch last 90 days of daily metrics.  
     - Daily cron → fetch yesterday’s metrics.  
   - Insert into `posts` + `daily_metrics` (raw metrics).

3. **Normalized metric service**
   - Compute `views_per_video`, `impressions_per_video`, `ctr`, `watch_time_per_view` per day.  
   - Expose via `/api/metrics/youtube/daily`.

### Dev B – Frontend: onboarding + charts

1. **Onboarding flow**
   - Auth (email/password or Supabase/Auth0).  
   - “Connect YouTube” button calling backend OAuth URL.  
   - Connection status indicator.

2. **Basic dashboard**
   - Graphs for YouTube views/impressions & `views_per_video`.  
   - Handle “no data yet” state.

3. **Landing page**
   - Simple marketing page explaining “Algorithm Crash Insurance” + early access form.

## Phase 2 – Baselines & anomaly detection (Week 3–5)

### Dev A – Detection engine

1. **Baseline calculation**
   - Cron job computing 30‑day & 90‑day EMAs + rolling stddev per creator/metric.  
   - Store in `metric_baselines` table.

2. **Anomaly detection**
   - Implement z‑score + multi‑day persistence logic.  
   - Simple changepoint detection (14‑day vs previous 30‑day mean).  
   - Rule Y1 for YouTube: create `alerts` when conditions met.

3. **Alert API**
   - `/api/alerts` list endpoint.  
   - `/api/alerts/:id` detail endpoint with raw numbers.

### Dev B – UX for anomalies

1. **Alert UI**
   - Display alerts list (type, date, severity).  
   - Detail view with:
     - Metric chart showing baseline vs actual.  
     - Non‑AI explanation from payload.

2. **Feedback controls**
   - “Accurate / Not accurate” toggle → POST to `/api/alerts/:id/feedback`.

3. **Refine UX**
   - Loading states, empty states, error toasts.
